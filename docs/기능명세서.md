# 썬드림 룰렛 기능명세서

## 1. 개요
본 문서는 실시간 웹소켓(Socket.IO) 기반 룰렛 이벤트 서비스의 핵심 아키텍처 및 구현된 기능 명세를 다룹니다.

## 2. 주요 기능 및 구현 방식

### 2.1 실시간 룰렛 엔진 (서버 주도 동기화)
- **당첨 조작 방지**: 서버(`comment_dart.py`)에서 최종 당첨 각도 및 당첨자를 100% 결정하여 클라이언트에 브로드캐스트합니다.
- **물리적 동기화**: 모든 접속 클라이언트(수십 대 이상)가 `start_game` 이벤트를 받아 소수점 수준까지 동일한 회전 애니메이션을 시청합니다.
- **가중치 기반 섹터 분배**: 참여자의 '티켓 수'에 비례하여 360도 원판 내 섹터 공간을 동적으로 할당하며 명단은 항상 가나다순으로 정렬됩니다.

### 2.2 하이브리드 DB 연동 (Supabase & SQLite)
- **Local SQLite**: 빠른 읽기/쓰기를 위해 메모리 및 로컬 파일 기반 DB를 운영합니다.
- **Cloud Supabase**: 영구 데이터 보존 및 모니터링 뷰어와의 실시간 연동을 담당하며, 서버 부팅 시 하이드레이션(Hydration)을 진행합니다.
- **비동기 동기화**: 메인 게임 루프를 방해하지 않도록 백그라운드 스레드에서 Supabase로 데이터 업서트를 처리합니다.

### 2.3 실시간 네이버 카페 댓글 스크래핑
- **V2 API 및 Selenium 폴백**: 네이버 카페 API를 우선 호출하되, 방어벽에 막히거나 누락 발생 시 Selenium 기반 브라우저가 개입합니다.
- **화이트리스트 기반 참여**: 특정 대상자 리스트에 존재하는 아이디만 필터링하여 게임에 투입합니다.

### 2.4 운영자 편의 및 제어 기능
- **시간 조작 기능**: '현재 시간'을 기준으로 +60초, +45초 버튼을 통해 즉각적인 목표 슈팅 시간 셋업이 가능합니다.
- **10초 쿨다운**: 당첨 결과 발표 직후 10초간 재시작을 방지하여 매끄러운 진행을 보장합니다.
- **모바일 맞춤 레이아웃**: 사은품 영역을 명단보다 위로 배치하고, 기기 환경에 맞춘 UI 동적 재배치를 지원합니다.
- **iOS 오디오 최적화**: 애플 정책을 우회하기 위한 전용 ON/OFF 토글 및 인터랙션 기반 오디오 Unlock 기능이 구현되어 있습니다.

### 2.5 Auto 연속 추첨 기능 (자동화)
- **자동 연속 추첨 (Auto Continuous Draw)**: 설정된 회차당 회전 시간(45초, 60초, 90초, 120초)과 10초 고정 휴식 대기 시간을 조합하여, 남은 사은품이 모두 소진될 때까지 무인으로 연달아 룰렛을 진행합니다.
- **실시간 대기 토스트**: 대기 시간 동안 `[Auto] N초 후 다음 추첨이 자동으로 시작됩니다.` 문구를 슛 타임 영역 하단에 보라색 토스트로 실시간 카운트다운 노출합니다.
- **예외 처리 및 안전 정지**:
  - 연속 추첨 중 언제든 일시정지 및 중지가 가능합니다 (룰렛 회전 중에는 비활성화, 10초 대기 시간 중에만 작동).
  - 룰렛 강제 정지(무효화) 발생 시 자동으로 연속 추첨도 취소됩니다.
  - 마지막 사은품이 추첨되어 당첨자가 확정되는 즉시 대기 카운트다운을 생략하고 자동화 모드를 바로 종료하며 사은품 소진 문구를 노출합니다.
